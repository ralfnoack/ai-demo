# Rector Coverage Cycle

Automatisierter Zyklus zur Steigerung der Typabdeckung mit Rector:

Wiederhole die folgenden Schritte, solange alle Tests fehlerfrei laufen:

1. TYPE_COVERAGE_LEVEL in @rector.php um +1 erhöhen und die aktiven Rector-Regeln dokumentieren:
   run php vendor/bin/rector list-rules > /docs/rector-rules.md
2. rector_optimize auf @/src ausführen und die Anzahl der geänderten Dateien notieren
3. Tests ausführen:
   run php vendor/bin/phpunit
4. Wenn Anzahl der geänderten Dateien > 0 ist UND die Tests fehlerfrei ausgeführt wurden, dann:
   git add .
   git commit -m "Steigerung der Typabdeckung auf LEVEL X"

Der Zyklus endet, sobald ein Test fehlschlägt.
